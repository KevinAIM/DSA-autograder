# M4 — Insertion Sort (reference for feedback)

Goal
Sort the input array in nondecreasing order (ascending). The function should modify the same array (in-place). It should work for edge cases: empty array, length 1, already sorted, reverse sorted, duplicates, negatives.

Core idea (what should happen)
We maintain a sorted prefix on the left. We take the next element (key) and insert it into the correct position inside the sorted prefix by shifting bigger elements one position to the right.

Expected control flow (pseudocode-style)
for j from 1 to n-1:
    key = array[j]
    i = j - 1
    while i >= 0 and array[i] > key:
        array[i + 1] = array[i]      # shift right
        i = i - 1
    array[i + 1] = key               # insert key at the “gap”

Important details
1) The final placement is array[i + 1] = key (not i - 1).
2) The while condition must prevent i from going below 0.
3) Shifting overwrites array[i+1], so you must save key before shifting.
4) It must handle duplicates correctly: using array[i] > key keeps equal values stable and avoids unnecessary shifts.
5) The algorithm should not return early unless the whole array is already processed.

Common student mistakes (patterns to check)
- Wrong final index when inserting key (array[i - 1] = key, array[j] = key, array[i] = key).
- Off-by-one in loop bounds (starting j at 0, or going to n instead of n-1).
- While condition missing i >= 0 or using the wrong comparator direction.
- Shifting the wrong way (array[i] = array[i+1]) or forgetting to decrement i.
- Forgetting to store key before shifting (key gets overwritten).
- Not actually sorting in-place (sorting a copy but not writing back).
- If other sorts (merge/quick) are part of the file: leaving stubs that break compilation or calling them incorrectly.
